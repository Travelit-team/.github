## 팀 컨벤션

### 깃 커밋 컨벤션

- init: 초기화  
- feat: 새로운 기능 추가  
- update: 기능 수정  
- fix: 버그 수정  
- docs: 문서 수정  
- style: 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우, linting  
- design: 레이아웃 수정, UX 또는 UI에 대한 커밋  
- refactor: 코드 리팩터링  
- chore: 빌드 업무 수정, 패키지 매니저 수정, 그 외 자잘한 수정에 대한 커밋   

### ***주의***
```
콜론(:)은 왼쪽에 붙여서 사용하자.
feat: 로그인 기능 구현 <---- feat 옆에 콜론을 붙이자! 
```
